# 🚀 Быстрый старт с FSD Project Starter Kit

## 📋 Что это?

Готовый шаблон для создания новых проектов с Feature-Sliced Design архитектурой с первого дня.

## ⚡ 5-минутная настройка

### Шаг 1: Клонируй или скачай шаблон

```bash
# Вариант A: Клонируй репозиторий
git clone <repository-url> my-new-project
cd my-new-project

# Вариант B: Скачай ZIP и распакуй
# Переименуй папку в my-new-project
cd my-new-project
```

### Шаг 2: Настрой проект

```bash
# Перемести конфиг Continue в правильное место
mv continue-config.yaml .continue/config.yaml

# Установи зависимости (выбери один вариант)

# Для Next.js:
npm install

# Для Vite:
npm install

# Для Create React App:
npm install
```

### Шаг 3: Настрой AI ассистент

#### Для Continue (VS Code):

1. Установи расширение "Continue" из Marketplace
2. Перезапусти VS Code
3. Открой папку проекта

#### Для Cursor:

1. Установи Cursor
2. Открой папку проекта
3. Конфигурация загрузится автоматически

### Шаг 4: Начни разработку с AI

Открой чат Continue/Cursor и используй команды:

```
/init              # Инициализировать проект
/shared button     # Создать кнопку
/feature auth      # Создать авторизацию
```

## 🎯 Первые шаги

### 1. Создай базовые компоненты

Используй промпты для создания:

- Кнопки, поля ввода, карточки
- Навигацию, шапку, подвал
- Базовые layout компоненты

### 2. Определи бизнес-сущности

Создай типы для:

- Пользователей, товаров, заказов
- Категорий, настроек, контента

### 3. Разработай основные фичи

Начни с ключевых сценариев:

- Авторизация и регистрация
- Главная страница и навигация
- Основной функционал приложения

## 🔧 Доступные команды

### Основные промпты:

- `/init` - Полная инициализация проекта
- `/shared [name]` - Создать shared компонент
- `/entity [name]` - Создать бизнес-сущность
- `/feature [name]` - Создать пользовательскую фичу
- `/widget [name]` - Создать виджет
- `/page [name]` - Создать страницу

### Вспомогательные:

- `/sass` - Настроить SASS архитектуру
- `/typescript` - Настроить TypeScript
- `/review` - Проверить архитектуру

## 📁 Структура проекта

После инициализации:

```
my-new-project/
├── src/
│   ├── app/           # Инициализация, провайдеры
│   ├── shared/        # Переиспользуемые компоненты
│   ├── entities/      # Бизнес-сущности
│   ├── features/      # Пользовательские сценарии
│   ├── widgets/       # Независимые блоки
│   └── pages/         # Страницы приложения
├── .continue/         # Конфигурация AI ассистента
└── docs/             # Документация
```

## 🎨 Стилизация

Проект использует SASS/SCSS модули:

- Переменные и миксины в `shared/styles/`
- CSS Modules для изоляции стилей
- Готовые responsive миксины

## 🔧 Настройка под свой проект

### Измени настройки в:

1. `package.json` - имя проекта, зависимости
2. `tsconfig.json` - пути, настройки TypeScript
3. `shared/styles/variables.scss` - цвета, шрифты
4. `.continue/config.yaml` - промпты под конкретный проект

## 🐛 Решение проблем

### Проект не запускается:

```bash
# Проверь зависимости
npm install

# Проверь TypeScript
npm run type-check

# Проверь линтинг
npm run lint
```

### AI ассистент не работает:

- Перезапусти VS Code/Cursor
- Проверь что `.continue/config.yaml` существует
- Убедись что расширение установлено

### Стили не применяются:

- Проверь импорты в SCSS файлах
- Убедись что SASS установлен
- Проверь конфигурацию сборщика
